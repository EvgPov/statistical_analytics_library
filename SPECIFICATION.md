# План разработки библиотеки статистического анализа (TDD-подход)
## Общие принципы
- Все функции принимают Iterable (любой итерируемый объект)
- При пустых данных выбрасывается StatisticsError
- При нескольких проходах по данным должна происходить загрузка в список с предупреждением, либо однопроходный алгоритм
- Для математических операций использовать модуль math
-Декораторы используют @functools.wraps

## План реализации по модулям
### 1. Модуль core.py (базовые статистические функции)
Функции:
1. Среднее арифметическое
    mean(data: Iterable[Number]) -> float
   - Вычисляется за один проход
   - Пустые входные данные -> StatisticsError
   - Тесты:
     - test_mean_list -> [1, 2, 3] -> 2.0
     - test_mean_generator -> (yelt 1, 2, 3) -> 2.0
     - test_mean_empty -> raises StatisticsError
     - test_mean_floats -> [1.5, 2.5] -> 2.0
     - test_mean_large_data -> не падает на 1 000 000 элементов
2. Медиана
    median(data: Iterable[Number]) -> float
   - Вычисляется за один проход
   - Пустые входные данные -> StatisticsError
   - Требует сортировки, поэтому загружаем в список, выдавая предупреждение
   - При четном количестве - среднее двух центральных элементов
3. Значение моды
    mode(data: Iterable[Number]) -> float
   - Если несколько мод, то возвращается первая встретившаяся (или список)
   - Пустые входные данные -> StatisticsError
   - Использует collections.Counter
4. Дисперсия
    variance(data: Iterable[Number], ddof: int = 0) -> float
   - Пустые входные данные или если ddof >= len -> StatisticsError
   - Реализуется онлайн-алгоритм (один проход) для эффективности
5. Стандартное отклонение (квадратный корень из дисперсии)
    variance(data: Iterable[Number], ddof: int = 0) -> float
   - Пустые входные данные или если ddof >= len -> StatisticsError
   
### 2. Модуль decorators.py (декораторы для обёртывания функций)
1. @timer
    - выводит в консоль время выполнения декорируемой функции в секундах (с точностью до миллисекунд).
    - применить к любой функции.
2. @logger
    - логирует вызов функции: имя, переданные аргументы (позиционные и менованные) и возвращаемое значение. Использует print или стандартный модуль logging.
    - применить к любой функции.
3. @validate_numeric
    - проверяет, что все аргументы-коллекции (списки, кортежи) содержат только числа (int или float). Если встречается нечисловой элемент, выбрасывается TypeError. Генераторы не проверяются (предполагается, что пользователь гарантирует корректность).
    - применить к функциям, которые ожидают числовые данные.
4. @memoize(maxsize=None)
    - кеширует результаты функции для заданных аргументов. Если maxsize задан, кеш ограничен указанным числом элементов (LRU-стратегия). Для реализации LRU можно использовать collections.OrderDict.
    - применит к вычислительно затратным функциям с хешируемым аргументам
